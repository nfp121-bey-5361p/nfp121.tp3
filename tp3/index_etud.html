Question 2.5

  public boolean equals(Object o) {
    if (o instanceof PileI) {
      PileI p = (PileI) o;
      return this.capacite() == p.capacite()
          && this.hashCode() == p.hashCode();
    } else
      return false;
  }

Reponse: La 1ere implementation de equals est correcte, car elle verifie le type actuel de l'instance o et de l'egalité de la capacité et de la valeur retournée par hashCode() de l'instance avec ceux de l'objet actuel.
p.hashCode() et this.hashCode() retournent la meme valeur si les 2 objets ont les memes état que hashCode() utilise pour calculer sa valeur de retour.

public boolean equals(Object o) {
    return this.toString().equals(o.toString());
  }
Reponse: ce code n'est pas correcte car:
  1. La vérification du type de o n'est pas presente.
  2. 2 piles de capacités differentes peuvent avoir le meme nombre d'élements et les memes valeurs
  3. Cas de o == null (NullPointerException)
